OOP原则

发表于 2017-07-22 | 分类于 读书笔记

开放-关闭原则

介绍

类应该对扩展开放，对修改关闭。 允许类容易扩展，在不修改代码的情况下，就可以应用于新的行为。这样的设计具有弹性，可以应对改变，可以接受新的功能来应对改变的需求。
典型应用：装饰者模式

依赖倒置原则

介绍

要依赖抽象，不要依赖具体类。不能让高层组件依赖低层组件，而且，不管高层组件还是低层组件，都要依赖抽象。相比于针对接口编程，不针对实现编程而言，更强调抽象性。

典型应用：工厂模式

最少知识原则（迪米特法则）

介绍

减少对象之间的交互。不要让太多类耦合在一起。

典型应用：适配者模式

单一职责原则

介绍

一个类干一件事情

里氏替换原则

介绍

所有引用基类的地方必须能透明地使用其子类的对象。使用抽象类/基类做为方法参数，具体使用时使用子类。

接口隔离原则

介绍

客户端不应该依赖它不需要的接口；类间的依赖应该建立在最小的接口上。接口应该建立单一接口，不要建立臃肿庞大的接口，尽量给调用者提供专门的接口，而非多功能接口。

其他书中提及原则

  • 找出应用中可能需要变化之处，独立出来，不要和那些不需要变化的代码混在一起
  • 针对接口编程，而不是针对实现编程
  • 多用组合，少用继承
